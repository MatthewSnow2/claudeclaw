Robotech TBS -- Full Codebase Review

Project Overview

A Civilization 6-inspired turn-based strategy game set in the Robotech/Macross universe. Built in Unity 2022 LTS (C#/MonoBehaviour). Single-player skirmish mode with hex grid, two factions (RDF vs Zentradi), a tech tree system, city founding/production, fog of war, and A* pathfinding. Currently a functional prototype using capsule primitives and Gizmo-based rendering.

1. `/tmp/robotech-review/Assets/Scripts/AI/AIController.cs`

  • Namespace/Class: Robotech.TBS.AI.AIController (MonoBehaviour)
  • Purpose: Orchestrates AI decision-making during the AI turn phase. Controls unit movement, combat targeting, city production, and tech research for the Zentradi faction.
  • Key Methods:
  • ExecuteAIPhase() -- Main coroutine: runs tech decision, city decisions, then unit actions sequentially with visual delays.
  • ProcessTechDecision() / SelectBestTech() / EvaluateTech() -- Picks the best available tech to research based on a weighted scoring system (critical path, era transition, military bonuses, cost, randomness).
  • ProcessCityDecisions() / SelectUnitToProduce() -- Queues unit production in idle AI cities by cloning existing non-settler unit types.
  • ProcessUnitActions() / ProcessSingleUnit() -- For each AI unit: try attack, then move toward nearest enemy/city, then try attack again.
  • TryAttackEnemy() / FindBestTarget() / EvaluateTarget() -- Target selection prioritizing low-HP, high-value, settler, and close targets.
  • TryMoveTowardObjective() / FindNearestObjective() / FindBestMoveToward() -- Pathfinding-based movement toward nearest enemy or city, with fallback to greedy adjacent-step movement.
  • Completeness: Fully implemented. Functional AI with tech, city, and unit management.
  • Dependencies: TurnManager, HexGrid, TechManager, CityManager, MapGenerator, UnitRegistry, CombatResolver, Pathfinder, MapRules, Unit, UnitDefinition, TechDefinition
  • Issues:
  • SelectUnitToProduce() only copies existing unit types -- if AI has no non-settler units, it builds nothing. No fallback to build from definitions.
  • Settler AI is explicitly skipped (if (unit.definition.canFoundCity) yield break), so AI never founds cities.
  • No strategic diversity (always attacks, never defends or retreats).
